① システム構成の概要

このシステムは、患者さんとスタッフが別々のデバイス（スマートフォン・PC）から同じ予約データにアクセスするために、Firebase Realtime Databaseを共有データベースとして使用します。

② Firebaseプロジェクトの作成

1
Firebase Consoleにアクセス
https://console.firebase.google.com をブラウザで開き、Googleアカウントでサインインします。
2
新しいプロジェクトを作成
「プロジェクトを追加」をクリック → プロジェクト名を入力（例：seiryo-shower）→ Googleアナリティクスは「有効にしない」で構いません → 「プロジェクトを作成」をクリックして完了まで待ちます。
3
Realtime Databaseを有効化
左メニューの「構築」→「Realtime Database」→「データベースを作成」をクリック。
ロケーション選択：「asia-southeast1（シンガポール）」を選ぶ（東京リージョンはRTDBでは使えません）。
セキュリティルール：「テストモードで開始」を選択 → 「有効にする」。
⚠️ テストモードは30日間のみ有効です。本番運用前に必ずステップ⑤のセキュリティルールに切り替えてください。
4
Webアプリを登録してconfigを取得
プロジェクトのトップ画面で「アプリを追加」→「ウェブ（</>）」をクリック。
アプリのニックネームを入力（例：shower-booking）→「Firebase Hostingの設定も行う」はチェック不要 → 「アプリを登録」。
次の画面に表示されるfirebaseConfigオブジェクトをコピーして保存しておきます。
// このような設定情報が表示されます（例） const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "seiryo-shower.firebaseapp.com", databaseURL: "https://seiryo-shower-default-rtdb.asia-southeast1.firebasedatabase.app", projectId: "seiryo-shower", storageBucket: "seiryo-shower.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef1234567890" };

③ HTMLファイルの設定（firebaseConfigを書き換える）

「シャワー予約_Firebase版.html」をテキストエディタで開きます（Windowsはメモ帳、Macはテキストエディット）。

1
ファイルを開く
HTMLファイルを右クリック →「プログラムから開く」→「メモ帳」（Windows）または「テキストエディット」（Mac）。
2
firebaseConfigを探して書き換える
Ctrl+F（Mac: Cmd+F）で「YOUR_API_KEY」を検索。見つかったら、以下のブロックをFirebaseコンソールからコピーした実際の値に置き換えます。
// 書き換え前（この7行を置き換える） const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT_ID.firebaseapp.com", databaseURL: "https://YOUR_PROJECT_ID-default-rtdb.asia-southeast1.firebasedatabase.app", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT_ID.appspot.com", messagingSenderId: "YOUR_MESSAGING_SENDER_ID", appId: "YOUR_APP_ID" };
3
保存して動作確認
ファイルを上書き保存（Ctrl+S）して、ブラウザでそのHTMLファイルを直接開いてみます。「接続中…」のローディング画面が表示され、すぐにシャワー予約画面になれば成功です。
💡 ファイルをブラウザで直接開いた場合（file://〜）でも動作確認はできますが、別のデバイスと同期させるにはWeb公開が必要です。次のステップへ進んでください。

④ Webホスティング（GitHub Pages）

患者さんがQRコードでアクセスできるよう、インターネット上にHTMLファイルを公開します。GitHub Pagesは無料で使えます。

1
GitHubアカウントを作成
https://github.com にアクセスして「Sign up」でアカウント登録（メールアドレスがあれば無料で作成できます）。
2
新しいリポジトリを作成
ログイン後、右上の「＋」→「New repository」。
Repository name：shower-booking
Public（公開）を選択 → 「Create repository」をクリック。
3
HTMLファイルをアップロード
作成したリポジトリ画面の「uploading an existing file」（または「Add file」→「Upload files」）をクリック。
「シャワー予約_Firebase版.html」をドラッグ＆ドロップ。
重要：ファイル名を index.html に変更してからアップロード（ファイル名フィールドで変更できます）。
「Commit changes」をクリック。
4
GitHub Pagesを有効化
リポジトリの「Settings」タブ →「Pages」（左サイドバー）。
Source：「Deploy from a branch」を選択。
Branch：「main」→「/(root)」→「Save」をクリック。
数分後、https://あなたのID.github.io/shower-booking/ でアクセスできるようになります。
📌 公開URLの例：https://seiryo-clinic.github.io/shower-booking/
このURLをQRコード化して患者さんに配布します。
⚠️ 更新時：HTMLファイルを更新した場合は、GitHubのリポジトリに再度アップロードし直してください（同じファイル名でアップロードすると上書きされます）。

⑤ データベースのセキュリティルール設定

テストモードの30日間期限が切れる前に、以下のルールに変更します。病院のLAN/Wi-Fiのみからアクセス可能にする設定も含まれます。

基本ルール（誰でも読み書き可能）

もっともシンプルな設定です。URLを知っている人のみアクセスできます。

{ "rules": { ".read": true, ".write": true } }
💡 Firebase Consoleの「Realtime Database」→「ルール」タブで上記のJSONに書き換えて「公開」ボタンをクリックします。
🔒 より高度なセキュリティが必要な場合：Firebase Authenticationを使ったユーザー認証の導入も可能ですが、患者さんのアカウント登録が必要になります。クリニック規模では上記の基本ルール＋管理者パスワード認証（すでに実装済み）で十分です。

⑥ QRコードの作成

1
無料QRコード生成サイトを使用
https://qr.quel.jp/ や https://www.qr-creator.com/jp/ などにアクセス。
GitHub PagesのURL（例：https://seiryo-clinic.github.io/shower-booking/）を入力してQRコードを生成。
PNG形式でダウンロードします。
2
掲示物の作成と設置
WordやPowerPointにQRコードを貼り付け、「シャワーのご予約はこちら」などの案内文を追加して印刷。
シャワー室の前、待合室、受付カウンターなど、患者さんの目に留まる場所に設置します。

⑦ スタッフの使い方

患者さんへの案内

手順内容
①スマートフォンでQRコードを読み取りサイトを開く
②初回のみ「お名前」と「患者ID・部屋番号」を設定する（デバイスに保存され次回以降は不要）
③希望の日付・時間・シャワー室を選んで「予約する」
④「あなたの予約一覧」でいつでも予約確認・キャンセルが可能

スタッフ（管理者）の操作

機能操作方法
管理画面を開く右上の「🔧 管理者」タブをクリック → パスワード入力（初期値：1234）
代理予約の登録「📊 当日の予約」タブ →「＋ 代理予約」ボタン
予約の削除各タブの予約一覧の「削除」ボタン
シャワー室を一日使用不可にする「🚫 使用停止設定」タブ → 該当日を選び「A室を使用不可」などをクリック
特定時間帯を停止「🚫 使用停止設定」タブ → 枠をクリックしてON/OFF
パスワード変更「🔑 パスワード変更」タブ
⚠️ パスワード変更について：管理者パスワードは各デバイスのブラウザに保存されます。パスワードを変更した場合、他のスタッフのPCでも変更後のパスワードが必要になりますので、変更後は必ず周知してください。

⑧ よくある問題と対処法

症状原因・対処法
「接続中…」から画面が変わらないfirebaseConfigの設定が間違っているか、インターネット接続がない。ブラウザのデベロッパーツール（F12）でエラーメッセージを確認。
「firebaseConfigを設定してください」と表示されるHTMLファイルのYOUR_API_KEY部分が書き換えられていない。手順③を再確認。
予約したのに他のデバイスに反映されない両方のデバイスがインターネットに接続されているか確認。FirebaseのRealtime DatabaseのConsoleでデータが書き込まれているか確認。
「データベースのアクセスが拒否されました」Firebaseのセキュリティルールの30日間テストモード期限が切れた可能性。手順⑤に従いルールを更新する。
スマホでレイアウトが崩れるブラウザのズーム設定をリセット（100%に）。または別のブラウザ（Chrome推奨）を試す。
QRコードを読み取れないQRコードが小さすぎる可能性。印刷サイズを大きく（最低3cm×3cm以上）する。

Firebaseコンソールでデータを直接確認する方法

予約データが正しく保存されているかは、Firebase Console（console.firebase.google.com）→ プロジェクト → Realtime Database → 「データ」タブで確認できます。

また、誤って登録された予約データはコンソール上で直接削除することも可能です。

お問い合わせ・更新について

  • システムの機能追加・改修が必要な場合はClaudeにHTMLファイルと変更内容を共有してください。
  • データのバックアップ：Firebase Consoleの「データのエクスポート」機能からJSONで出力可能です。
  • Firebaseの無料枠を超えた場合は有料プランへの移行が必要です（通常のクリニック規模では不要）。
